Presentación del curso

En este curso aprenderás los fundamentos de Angular. Comenzaremos con una introducción a la estructura de una aplicación de Angular y cómo crear y usar componentes. Luego, cubriremos el enlace de datos y el manejo de eventos, el enrutamiento y la navegación entre páginas, y el uso de servicios y observables para trabajar con datos. Así mismo, serás capaz de trabajar con diferentes componentes, como directivas, pipes y módulos. Al final del curso, tendrás las habilidades necesarias para crear tus propias aplicaciones de Angular desde cero y comprenderás los conceptos fundamentales detrás de este poderoso framework.

Objetivos

  • Comprender los conceptos básicos de Angular como la estructura de una aplicación de Angular, los componentes, el enrutamiento, la navegación, el enlace de datos y los servicios.
  • Familiarizarse con el entorno de desarrollo de Angular
  • Desarrollar habilidades prácticas para crear aplicaciones web de Angular, componentes, enrutamientos, enlace de datos, entre otros.
  • Conocer las buenas prácticas de desarrollo de Angular, como la organización del código, la modularidad, y la estructuración de los componentes y servicios.
  • Prepararse para cursos más avanzados, en los que se profundiza en temas más complejos y avanzados, como la animación, la seguridad y la optimización de la aplicación.

MÓDULO 1. INTRODUCCIÓN A ANGULAR

¿Qué es Angular?

  • Angular es un framework de desarrollo de aplicaciones web de código abierto y mantenimiento de la comunidad
  • Se destaca por su estructura basada en componentes y su enfoque en la creación de aplicaciones SPA (Single Page Aplication), además de ofrecer una estructura sólida y simplificada para su desarrollo.
  • Utiliza TypeScript para construir aplicaciones del lado del cliente, con un patrón de diseño Modelo (datos y la lógica de la aplicación), Vista (interfaz de usuario) y Controlador (conduce las interacciones entre M-V)(MVC)
  • Posee modularidad, es multiplataforma y posee una gran comunidad. ADemás, se compara comúnmente con otros frameworks como React y Vue.js en términos de rendimiento y características.
  • AngularJS fue lanzado por Google en 2010, dirigido por Miško Hevery y Adam Abrons
  • Fue popular por su enfoque en el desarrollo del lado del cliente y la manipulación del DOM.
  • La reescritura completa de AngularJS, se lanzó en 2016, con un enfoque en la mejora del rendimiento y la escalabilidad
  • Desde entonces, Angular ha seguido evolucionando con nuevas versiones y actualizaciones periódicas.

Instalación de sofware para uso de Angular

OS Linux

  1. Abre una terminal de Linux.

  2. Instala Node.js con el siguiente comando: 
    sudo apt-get update
sudo apt-get install nodejs
  3. Instala npm (Node Package Manager) con el siguiente comando: 
    sudo apt-get install npm
  4. Instala Angular CLI (Command Line Interface) globalmente usando npm: 
      sudo npm install -g @angular/cli
  5. Verifica la instalación escribiendo el siguiente comando: 
    ng version
  6. Descargar e instalar VSC para Debian, Ubuntu, Red Hat, Fedora, SUSE, etc. mediante archivos .deb, .rpm, .tar.gz, desde la página https://code.visualstudio.com/
  7. Ahora instalamos TS mediante el comando 
     npm install -g typescript

OS Windows

  1. Descarga e instala Node.js desde el sitio web oficial: nodejs.org. Asegúrate de descargar la versión LTS (Long Term Support) para obtener una versión estable.
  2. Abre una ventana de comandos (Command Prompt) o PowerShell.

  3. Instala Angular CLI (Command Line Interface) globalmente usando npm (Node Package Manager) con el siguiente comando: 
    npm install -g @angular/cli
  4. Verifica la instalación escribiendo el siguiente comando: 
    ng version
  5. Descargar e instalar VSC para Windows 10 y 11 desde la página https://code.visualstudio.com/
  6. Ahora instalamos TS mediante el comando 
     npm install -g typescript

macOS

  1. Abre la terminal en macOS.
  2. Instala Homebrew si aún no lo has hecho. Puedes utilizar el siguiente comando en la terminal:

  3. Después de instalar Homebrew, puedes instalar Node.js y npm con el siguiente comando en la terminal:

  6. Descargar e instalar VSC para macOS 10.15+ desde la página https://code.visualstudio.com/

Familiarizándose con TS, HTML y CSS

TypeScript (TS) es un lenguaje de programación de código abierto desarrollado por Microsoft. Es un superconjunto tipado de JavaScript que se compila en código JavaScript simple. La principal característica de TypeScript es la adición de un sistema de tipos estáticos opcional, lo que significa que permite al desarrollador agregar tipos a las variables y funciones. Estos tipos pueden ser verificados por el compilador en tiempo de compilación para detectar posibles errores y mejorar la calidad del código.

Además del sistema de tipos estáticos, TypeScript incluye características adicionales que no están presentes en JavaScript estándar, como las interfaces, las clases y los módulos, lo que facilita la estructuración y organización de código para proyectos más grandes. Al usar TypeScript, los desarrolladores pueden aprovechar las características de ECMAScript más recientes (ES6, ES7, etc.) mientras se benefician de la verificación estática de tipos y las características de orientación a objetos.

La idea principal detrás de TypeScript es mejorar el desarrollo de aplicaciones JavaScript a gran escala al proporcionar herramientas para capturar errores comunes y mejorar la productividad del desarrollador. TypeScript se puede utilizar en una variedad de entornos de desarrollo, desde aplicaciones front-end hasta back-end y aplicaciones web en general. Además, es compatible con muchos editores de código y entornos de desarrollo, lo que lo convierte en una opción popular para muchos desarrolladores y equipos de desarrollo.

Los pasos para crear, compilar y ejecutar un código .ts en VSC son los siguientes:

  1. En VSC crea un nuevo archivo con la extensión .ts.
  2. Escribe tu código TypeScript en el archivo.
  3. Para compilar el archivo TypeScript puedes usar el siguiente comando en la terminal de VSC: 
     tsc nombre_del_archivo.ts
  4. Para ejecutar el archivo TypeScript puedes usar el siguiente comando en la terminal de VSC: 
     node nombre_del_archivo.js
  5. Para compilar y ejecutar el archivo TypeScript puedes usar el siguiente comando en la terminal de VSC: 
     tsc nombre_del_archivo.ts && node nombre_del_archivo.js

MÓDULO 2. COMPONENTES

Creación de componentes

Los componentes son los bloques de construcción fundamentales de una aplicación en Angular. Son responsables de controlar una parte de la interfaz de usuario y pueden contener datos, lógica y vistas.
Para crear un nuevo componente en Angular, existen dos métodos:
  1. Método Manual, donde hay que efectuar la construcción y configuración de los componentes.
  2. Método Automático, con Angular CLI se puede generar de forma sencilla componentes preconfigurados.

Método Manual

  1. Primero dirígete a la carpeta scr/app/"nombre_app"
  2. Dentro de la carpeta crea 3 archivos:
    • "nombrecomp".component.ts
    • "nombrecomp".component.html
    • "nombrecomp".component.css
  3. Dentro del archivo .ts carga el módulo de Component, el @Component y la clase.
  4. Importa el nuevo módulo en app.module.ts: 
    import{"nombrecomp"} from "./nombrecomp/nombrecomp.component";
    y registra el nombre de la clase en declarations:
    NombrecompComponent
Donde hay que cambiar "nombrecomp" por el nombre que emplearás.

Método Automático

Para usar este método primero dirígete a la carpeta scr/app/"nombre_app", y luego utiliza el siguiente comando en la terminal: 
ng generate component "nombre-del-componente"

También se puede emplear la instrucción de forma compacta:

 ng g c "nombre-del-componente"
Esto creará automáticamente una nueva carpeta con los archivos necesarios para el componente. Ya solo debes dar de importar y declarar la clase en app.module.ts .

Vinculación (Binding)

Se refiere a la forma en que los datos fluyen entre el componente y la vista en una aplicación. Angular proporciona varios tipos de "bindings" que facilitan la interacción dinámica entre el modelo de datos y la interfaz de usuario. Estos "bindings" permiten que los datos se mantengan sincronizados y actualizados en tiempo real entre la vista y el componente.

Interpolación (Interpolation)

Este es un tipo de vinculación (binding) unidireccional que permite la interpolación de valores de propiedades del componente directamente en la vista. Se utiliza con la sintaxis de doble llave {{ valor }} en la plantilla HTML para mostrar datos dinámicos.

Cambiemos el título de la aplicación, el cual es fijo, por un texto dinámico mediante interpolación. Para esto, en el .ts principal, agregamos en la clase: 
Titulo="Demanda de Medicamentos IMSS";
Mientras que en el .html principal, borremos el título que colocamos y pongamos: 
  <h1> {{Titulo}} </h1>

Vinculación de Propiedad (Property Binding)

Permite asignar dinámicamente valores a atributos de elementos HTML o a propiedades de componentes utilizando la sintaxis de corchetes [propiedad]="valor" en la plantilla HTML. Esto permite actualizar las propiedades de los elementos HTML o los componentes en función de cambios en el modelo de datos.

Vinculación de Evento (Event Binding)

Permite vincular eventos del DOM, como clics del ratón o pulsaciones de teclas, a métodos definidos en el componente. Se utiliza con la sintaxis de paréntesis (evento)="manejador()" en la plantilla HTML para llamar a funciones o métodos del componente cuando ocurren eventos en la vista.

El término "DOM" significa "Document Object Model" (Modelo de Objetos del Documento) y es una representación de la estructura del documento HTML que proporciona una interfaz para acceder y manipular dinámicamente el contenido, la estructura y el estilo del documento. Es decir, el DOM es una representación de cómo se organiza y estructura un documento HTML en forma de árbol.

Vinculación Bidireccional (Two-way binding)

Permite la sincronización de datos en ambos sentidos, es decir, desde el componente hacia la vista y viceversa. Se utiliza con la directiva [(ngModel)] para enlazar un elemento de entrada de formulario, como un campo de texto, con una propiedad del componente. Los cambios en el modelo de datos se reflejan inmediatamente en la vista y viceversa.

La expresión ngModel es una directiva de Angular de tipo Atributo. Además de este tipo existen directivas Estructurales y otras endémicas de Angular, cómo lo podremos ver más adelante.

Comunicación entre componentes

  1. @Input{}
  2. @Output{}
El uso del primer decorador sucede cuando queremos que un componente padre se comunique con un componente hijo. El segundo sucede cuando se quiere el caso opuesto, el componente hijo se comunique con el componente padre.

Componentes Dinámicos

Un componente dinámico es un tipo de componente que se crea, se carga o se destruye en tiempo de ejecución, en lugar de estar presente en el código fuente en el momento de la compilación. Esto permite la creación de componentes de forma dinámica en respuesta a eventos o condiciones específicas durante la ejecución de la aplicación.

Ciclo de vida de un componente

No es necesario implementar todos (o ninguno) de los ganchos del ciclo de vida, solo los que necesites.
Hook method Purpose Timing
ngOnChanges() Responder cuando Angular establece o reinicia propiedades de entrada vinculadas a datos. El método recibe un objeto SimpleChanges con los valores actuales y anteriores de las propiedades. Llamado antes de ngOnInit() (si el componente tiene entradas vinculadas) y cada vez que una o más propiedades de entrada vinculadas a datos cambian.
ngOnInit() Inicializar la directiva o componente después de que Angular muestra por primera vez las propiedades vinculadas a datos y establece las propiedades de entrada de la directiva o componente. Consulte los detalles en Inicialización de un componente o directiva en este documento. Llamado una vez, después de ngOnChanges() por primera vez. ngOnInit() sigue llamándose incluso cuando ngOnChanges() no lo hace (lo cual es el caso cuando no hay entradas vinculadas a la plantilla).
ngDoCheck() Detectar y actuar sobre cambios que Angular no puede o no detectará por sí solo. Consulte los detalles y el ejemplo en Definición de detección de cambios personalizada en este documento. Llamado inmediatamente después de ngOnChanges() en cada ejecución de detección de cambios y inmediatamente después de ngOnInit() en la primera ejecución.
ngAfterContentInit() Responder después de que Angular proyecta contenido externo en la vista del componente, o en la vista en la que se encuentra una directiva. Consulte los detalles y el ejemplo en Respuesta a cambios en el contenido en este documento. Llamado una vez después del primer ngDoCheck().
ngAfterContentChecked() Responder después de que Angular verifica el contenido proyectado en la directiva o componente. Consulte los detalles y el ejemplo en Respuesta a cambios en el contenido proyectado en este documento. Llamado después de ngAfterContentInit() y en cada ngDoCheck posterior.
ngAfterViewInit() Responder después de que Angular inicializa las vistas del componente y las vistas secundarias, o la vista que contiene la directiva. Consulte los detalles y el ejemplo en Respuesta a cambios de vista en este documento. Llamado una vez después del primer ngAfterContentChecked().
ngAfterViewChecked() Responder después de que Angular verifica las vistas del componente y las vistas secundarias, o la vista que contiene la directiva. Llamado después de ngAfterViewInit() y en cada ngAfterContentChecked posterior.
ngOnDestroy() Limpieza justo antes de que Angular destruya la directiva o componente. Anule la suscripción a Observables y desconecte los controladores de eventos para evitar pérdidas de memoria. Consulte los detalles en Limpieza en la destrucción de la instancia en este documento. Llamado inmediatamente antes de que Angular destruya la directiva o componente.
